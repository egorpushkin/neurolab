Scripting Environment 1.0 - Developer Guide
Scripts Language Overview
The Lua Engine used to power realtime scripting machine.
The application extend the functionality of Lua by adding three custom packages:
  • Log Processing
  • Data Processing
  • Network Processing
Each package includes functions for operating with corresponding objects (Log Processing - ILogProvider; Data Processing - IDataFile, IDataFactory; Network Processing - INetworkProcessor, IElementsCollection, IElement, IDataFlow, IConnectionsFactory, IDataConnector, ILayerControl).

Remarks
All additional packages, that were addded to interpreter (Log Processing, Data Processing, Network Processing), have functions which return light userdata objects. It allows the environment to pass to the script pointers that refers to data objects created by application (data files, data factories, elements of built network, ect.). These userdata objects are called just 'Object' in package overview departments when telling about API function declaration. Here are some other definitions which were used to determine variable type:
Type name Description
Void Can be used only as function return value type. It means that function returns no value.
String This expression tells that data must be of the string type.
Integer This expression tells that function parameter must be whole number or function returns integer value.
Float This expression tells that function parameter data must be fractional number.
Boolean Boolean value required or returned by a function.
Constant You must specify here one of the constants which are predefined. It can be any of network element types or type of connections factory.
