Data Processing SDK - Object Model
Object model built for Neuro Laboratory is based on the IObject interface.
Hierarchy Chart
Most of these sections are implemented by internal Neuro Laboratory components. You will not operate with these modules even during writing you own libraries which extend application functionality. But enumeration and quick tour is importand for understanding Neuro Laboratory architect and development process.
Application Environment Section
This IAppController interface was implemented by the environment to give an opportunity of passing control interfaces (such as INetworkProcessor, ect.) to external application components (plug-ins, trainers).
Neuro Factory Section
This section implements functionality of neural network processing control interfaces:
  • INetworkProcessor - performs data flow operations over built neural network.
  • IElementsCollection - gives an opportunity to enumerate network elements.
  • INetworkVerifier - performs network verification.
Network Importing Section
This section is implemented by external library NetworkImport.dll (included in Network Exporter Tool package) and contains functionality for importing Neuro Laboratory project files in users' applications.
Components Section
This section implements functionality for connecting external application conponents.
Plug-ins Subsystem Section
This section is implemented to give an opportunity of connection external libraries which extends functionality of Neuro Laboratory.
Objects Messaging Section
ILogProvider interface can be implemented by any object. It gives an opportunity to create and display a log of performed operations.
Network Output Section
This section contains control interfaces for manipulating neural network output.
Network Trainers Section
ITarinersContainer is implemented by the environment to give an opportunity of connecting external libraries which are called trainers. The main idea of such a library is implementing training algorithm.
Storage and Data Processors Sections
This sections are directly connected with data processing.
  • Storage - This section contains interfaces that can be implemented by any object which requires presence of serialization operations.
  • Data Processors - This section contains interfaces which are implemented by main data processing objects.
