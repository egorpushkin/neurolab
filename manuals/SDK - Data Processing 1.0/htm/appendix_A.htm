Appendix A - Images Data File Format
This files are used in Neuro Laboratory for containing data for training and processing neural networks (*.nldfile). File data is a number of one after another written images (input than output).
Format Description
Data file conatins the following fields:
Field Type Field Name Number of Entries Field Size Description
Compulsory fields (file header)
CString File Index 1 null-terminated string Contains string that identifacates a file as Data Images File - "#NL_DATA_FILE".
CString File Version 1 null-terminated string Contains file version - "1.0.0.0a".
int InputSymbolsCount 1 sizeof(int) Contains a number of input symbols in a file.
int InputSymbolLength 1 sizeof(int) Contains a length of input symbol.
int OutputSymbolsCount 1 sizeof(int) Contains a number of output symbols in a file.
int OutputSymbolLength 1 sizeof(int) Contains a length of output symbol.
Optional fields (file body)
CDataFactory InputSymbol i InputSymbols Count sizeof(bool) [+ sizeof(int) +
sizeof(float) * InputSymbolLength]		 Contains information about input symbol.
CDataFactory OutputSymbol i OutputSymbols Count sizeof(bool) [+ sizeof(int) +
sizeof(float) * OutputSymbolLength]		 Contains information about output symbol.
Note Application always writes/reads strings in ANSI charset.
Usage
Sample source for loading such a file: 
// csFileName contains a file name of existing data file
CDataFile mFile;

bool bSuccess = mFile.LoadFile(csFileName);
// LoadFile method returns true in case of successful loading
Note In case of wrong file contents or reading error (when LoadFile method returns false) all previosly stored in the CDataFile object data would be lost. It happens because CDataFile object releases it's data before starting reading data file.
